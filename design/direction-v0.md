## 0. 출발점 — 정체성 선언

DOCK은 **편집권의 벽**을 깨는 도구다. 학생의 발화가 외벽 풍경의 일부가 되되, *그 형식까지 학생이 결정한다*는 게 절대 원칙. 시스템은 추천하지 않는다.

**우리는 다음을 거부한다:**
- ❌ AI/알고리즘이 톤 추천
- ❌ 상투적인 글쓰기 앱 흐름 ("쓰기 → 폼 채우기 → 완료")
- ❌ "감정 슬라이더" 같은 추상 표현 강요
- ❌ 코퍼레이트 톤, 파스텔, 과한 화려함
- ❌ 깔끔하지만 영혼 없는 미니멀리즘

**우리는 다음을 추구한다:**
- ✓ *만지작거리며 발견*하는 탐색
- ✓ 의례감(rituality)과 명상적 무게
- ✓ 한국 캠퍼스의 야간 외벽이 가진 정서
- ✓ GT Mechanik/Grilli의 *정신*(한 글자에 정성), 다만 색·그래픽은 우리 어휘
- ✓ Karel Martens의 *순색 자신감* 과 광학적 색 혼합
- ✓ 인쇄·아날로그 물성(Riso) 의 무게
- ✓ 11개 발화가 외벽에서 합창하는 풍경감

## 1. 흐름 결정 — Option Z

기존 9 phase 중 4(작성)와 5(톤 결정)를 **통합된 한 화면**으로 합친다.

```
NFC 첫 태깅 → Z 통합 화면 (쓰기 + 톤 결정 동시) → 맡기기
              → 재태깅 안내 → NFC 재태깅 → 외벽 도착
```

**Z 통합 화면의 핵심:**
- 거대한 단일 글자(사용자가 친 마지막 글자)가 화면 절반 이상을 점령
- 그 글자가 *라이브 글리프 테스터*로 작동 — 컨트롤을 만지면 즉시 변형
- 작은 전체 메시지 미리보기로 60자가 어떻게 보일지 확인
- 텍스트 입력은 작은 영역에서, 화면의 주인공은 *발화 자체의 형태*

**Grilli 정통성:** 폰트 디자이너가 한 자에 시간을 들이듯, 발화자도 *자기 발화의 형태에 시간을 들인다*. 글쓰기 ≠ 폼 채우기. 의례다.

**구현:** `?mode=z` 쿼리로 격리된 프로토타입 (현 main 흐름과 공존). 정전(canon)으로 승격할지는 사용자 검증 후 결정.

## 2. 색 시스템 — 4 큐레이팅 무드

| 무드 | 의도 | bg | text | graphic |
|---|---|---|---|---|
| **NIGHT 밤** | 집중, 의례, 톤 발견의 명상 | `#0A0A0A` | `#F2F2F2` | `#FFB800` |
| **PRINT 인쇄** | 선언적, 진지, "찍힘"의 흔적 | `#F5F4ED` | `#1A3A8C` | `#E84545` |
| **DAY 낮** | 활기, 합창, 풍경 합류 | `#FFFAE3` | `#0A0A0A` | `#FF1493` |
| **CLASSIC 기본** | 출발점, 익숙함, Mechanik 헌정 | `#FCE7F3` | `#C2185B` | `#8B9A1B` |

**원칙:**
- 채도는 풀(full) — 파스텔 거부
- 4개로 시작. 추가는 나중에 의도 부여 후 (무드별 정체성이 흐려지지 않게)
- COLOR 버튼은 *순환* — 무작위 아닌 정주행. 결정이 의례적
- 명도 대비 4.5:1 이상 보장 (가독성)
- 그래픽 SVG는 *반투명*화 검토 — Karel Martens 식 광학 혼합 시도 (다음 단계)

## 3. 모션 어휘

| 상황 | 어휘 | 출처 |
|---|---|---|
| 정지/대기 (외벽에 자리잡은 메시지) | 인쇄적 반복/에코 | R01 Powers of Ten |
| 등장/변형 (메시지가 외벽에 도착) | 비틀린 원근, 깊이 회전 | R02 워드마크 모션 |
| 톤 발견 (글자 만지작) | 가변폰트 축의 즉각 반응 | R03 Grilli tester |
| 줄 펄스 (베이스라인) | 굵기 변화 → 미세한 형태 변화 동반 | R02 + R03 |

**모션은 모두 "느리고 절제됨".** 글리치/폭발/패닉 없음. 외벽이 *숨쉬는* 감각.

## 4. 타이포그래피

- **폰트 3종 (현행 유지):** D2Coding(모노) / Pretendard(고딕) / Noto Serif KR(명조)
- **글자가 주인공.** 데코가 아니라 화면을 점령. SIZE 한도(현 72px)는 작음 — Z 모드에선 단일 글자가 200–320px
- **줄 펄스 = 외벽의 호흡.** 베이스라인. 끄거나 조절 불가.
- **컨트롤 라벨은 모노 약어** (FONT/WGHT/SLNT/TONE/SIZE) — 폰트 specimen 관습

## 5. 컨트롤 어휘 (Option Z 단순화)

| 축 | 옵션 | 비고 |
|---|---|---|
| FONT | 모노 / 고딕 / 명조 | 3 |
| WGHT | THIN / RGLR / BOLD / BLAK | 6 → **4** (간략화) |
| TONE | 장 / 평 / 평체 | 3 |
| SLNT | UPRT / OBLQ | 2 |

| COLOR | 4 무드 순환 | (별도 행) |
| STAR | 5 그래픽 + 무작위 끄기 | 현행 유지 (그래픽 어휘 재설계는 다음 라운드) |

## 6. 안티-패턴 (절대 하지 말 것)

- 🚫 글자가 데코로 작아지는 것 — DOCK에서 글자는 *항상* 주인공
- 🚫 음성/AI/기계 학습으로 톤 *추천* — 입력으로만 사용, 추천 금지
- 🚫 10개 이상의 팔레트 — 결정 부담을 피한다는 명목으로 의도를 흐림
- 🚫 부드러운 그라데이션, 코퍼레이트 블루, 파스텔
- 🚫 폼/카드/위젯의 시각 어휘 — DOCK은 *외벽*이지 *대시보드*가 아니다
- 🚫 GT Mechanik 그래픽 5종을 그대로 두는 것 — 다음 라운드에서 DOCK 고유 그래픽 어휘 정립
