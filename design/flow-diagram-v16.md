# DOCK Allocated Flow Diagram — v16

> 2026-05-19 작성. v15(2026-05-12) 대비 변경/추가/확인 필요 항목 정리.
> v15는 *3-앱 아키텍처*(Input/Server/Output)를 못박았는데, 우리는 그동안
> **Input + Web-Output**까지 구현했고, **Server(R2 Pi)를 우회**하고
> **Firebase**를 데이터 백본으로 썼다. 그 결정의 영향을 짚는다.

---

## 0. 변경 요약 한 줄

```
v15: 사용자 폰 PWA + R2 라즈베리파이 + 외벽 프로젝션 (3-앱)
v16: 사용자 폰 PWA + Firebase + Web 시뮬레이션 (2-앱·1-클라우드 + 미래의 물리 R2)
```

R2/물리 외벽은 *별도 트랙*으로 분리하고, *디지털만으로 흐름이 완성*되게 했다.

## 1. 아키텍처 — v15 vs v16

### v15 (계획)
| 컴포넌트 | 역할 |
|---|---|
| **Input** PWA (폰) | 작성·톤·캡처·로컬 저장 |
| **Server** R2 라즈베리파이 | NFC reader, 거치대 센서, NeoPixel, 메시지 큐, WebSocket broadcaster, 7일 수명 관리 |
| **Output** 호스트 (Chrome kiosk) | 풍경 렌더 (11개 메시지, 위로 흐름) |
| 통신 | HTTP POST / WebSocket / LAN (R2 핫스팟 또는 캠퍼스 WiFi) |

### v16 (현재)
| 컴포넌트 | 역할 |
|---|---|
| **Input** PWA (폰) | 작성·톤·송신. 캡처/로컬 저장은 미구현 |
| **Cloud** Firebase Firestore | 메시지 저장·실시간 동기화 (onSnapshot). 큐 관리 X — 시간 기반 필터링만 |
| **Output** Web (`/wall`) | 풍경 렌더 (수평 다중 트랙, 동적 N개) — 브라우저에서 작동. Pi/projector는 *아직 없음* |
| **Mock** `?mock=1` | 개발용 localStorage 대체 백엔드 |
| 통신 | HTTPS / WebSocket (Firestore SDK long-polling 강제) |

### 의미
- **R2 라즈베리파이 → 클라우드(Firebase)로 이동.** 물리 dock이 없으니 NFC·LED·압력 센서 전부 *URL 파라미터*로 시뮬레이션 (`?stage=enter` / `?stage=submit`).
- **2단계 도킹 → 1단계 + 시뮬 버튼.** "4cm 더 깊이 밀어넣기"는 현재 *(개발) 바로 보내기* 버튼.
- **7일 수명** → Firestore에 영구 저장하되 `/wall`이 *읽을 때 필터링* (createdAt + 7d).
- **외벽 프로젝션** → 아직 *물리 디스플레이 없음*. `/wall`이 그 미리보기.

## 2. Phase별 변경사항 (Input app 중심)

| Phase | v15 의도 | v16 현재 | 상태 |
|---|---|---|---|
| **01 발견** | 캠퍼스에서 두 빛(외벽 + R2 돔) 인지 | *해당 없음* (물리 R2 없음). 사용자가 NFC URL로 직접 진입 또는 홈에서 시작 | 🟡 물리적 phase, 가상 |
| **02 접근** | R2 측면·뒤로 다가가 NFC 마크 인지 | *해당 없음* | 🟡 물리적 |
| **03 접속** | R2 후면 거치대에 폰 안착 → 진동 + 자동 웹앱 실행 | URL 진입 (`?stage=enter`). 진동/자동 실행 미구현 | 🟡 NFC 태그 굽기 전 시뮬 |
| **04 작성** | 60자 입력 + 줄바꿈 + 60자 카운터 + "다음" | **Z1**과 통합 (작성+자형 한 화면) | ✅ 구현, 단 v15의 *별도 Phase 04*는 폐기 |
| **05 톤 결정** | 폰트/굵기/기울기/크기 슬라이더 + 가변폰트 미리보기 + 줄 펄스 | **Z1 자형 + Z2 색·그래픽**으로 *둘로 쪼갬* | ✅ 구현. v15의 단일 phase 5는 **두 단계로 분할** |
| **05b 도킹 안내** | 2단계 깊은 도킹 (4cm 더) 안내 | "다시 거치대에 올려주세요" + (개발) 바로 보내기 버튼 | 🟡 NFC 태그 전 시뮬 |
| **06 도킹 완료** | "외벽에 떠올랐다" 인지 + 외벽 향해 돌아섬 | **외벽에 합류했어요** 카드 + 약속 (12초 + 7일 + 삭제 안내) | ✅ 구현, 캡처는 미구현 |
| **07 마주봄** | 본인 메시지 외벽에서 응시 (10-60초) | `/wall`의 **emphasis mode 12초**가 이 phase의 디지털 등가물 | 🟡 의도된 매핑이지만 *물리 외벽*과 다름 |
| **08 합류** | 5초 페이드 후 메시지가 풍경 흐름에 합류 | `/wall` emphasis 모드의 settle 단계 (last 2s scale out → 트랙으로) | ✅ 구현 |
| **09 떠남** | 사용자가 떠남 + 캡처 저장 + 인스타 공유 옵션 | 캡처/공유 **미구현**. 사용자가 그냥 다시 쓰기 또는 종료 | 🔴 미구현 |
| **장기 후속** | 외벽 높이 = 메시지 나이 (위로 떠오름) → 80% 도달 시 페이드 → 사라짐 | *수평 트랙* 모델로 변경. 7일 후 표시 안 함 (DB엔 남음, 자동 삭제 X) | 🟡 수직 → 수평으로 변경 |

## 4. 시트 7 (System Architecture) — v16 갱신

```
┌─────────────────────────┐         ┌─────────────────────────┐
│  [INPUT APP]            │         │  [OUTPUT WEB / 미래 PI] │
│  사용자 폰 PWA          │         │  /wall 또는 Chrome      │
│                         │         │  kiosk on Pi 5 (미래)   │
│  · 홈 / Z1 / Z2 / 06    │         │                         │
│  · Pretendard/Orbit/    │         │  · 수평 6 트랙 풍경     │
│    Noto Serif KR/       │         │  · 12s emphasis        │
│    Sunflower            │         │  · 24개 / 7일 필터     │
│  · 5 무드 + 5 그래픽    │         │  · 실시간 onSnapshot   │
│  · 가변폰트 + 줄 펄스   │         │                         │
└──────────┬──────────────┘         └──────────────▲──────────┘
           │ Firestore Web SDK                     │ Firestore Web SDK
           │ (HTTPS long-polling)                  │ (onSnapshot)
           ▼                                       │
┌─────────────────────────────────────────────────┴────────────┐
│  [CLOUD] Firebase Firestore                                  │
│  · messages 컬렉션 (text, tone, createdAt)                   │
│  · 보안 규칙 (현재 wide-open, 출시 전 좁힘)                  │
│  · firebase-tools CLI로 룰 배포                              │
└──────────────────────────────────────────────────────────────┘

[FUTURE — 물리 설치 단계에서 추가]
┌─────────────────────────┐
│  [SERVER] R2 Pi 5       │
│  · NFC reader (PN532)   │
│  · 거치대 압력/스위치   │
│  · NeoPixel LED         │
│  · Firestore Web SDK    │  ← 폰과 동일하게 클라우드 의존
│    (or local cache)     │
└─────────────────────────┘
                            ↓ HDMI
                          [프로젝터 / 외벽]
```

물리적 R2와 외벽은 *추가될 뿐 아키텍처를 바꾸지 않음*. Pi가 Firestore에 read/write하는 클라이언트가 되는 셈.

## 5. 시트 8 (Data Contract) — v16 갱신

### 메시지 객체 — 현재 스키마

```ts
interface StoredMessage {
  id: string;               // Firestore docId
  text: string;             // max 60자
  tone: ToneState | null;
  createdAt: number;        // ms epoch (서버 timestamp → toMillis)
}

interface ToneState {
  font: 'mono' | 'gothic' | 'myeongjo' | 'song';  // ← v15 +1
  tone: number;       // 0.7 | 1.0 | 1.3 (scaleX)
  wght: number;       // 100 | 400 | 700 | 900
  slnt: number;       // 0 | -8
  size: number;       // 40-180
  paletteIdx: number; // 0..4 (5 무드)              ← v15 0..9
  graphicIdx: number; // -1 (off) | 0..4 (자모 골격) ← v15 sphere/dots/grid/concentric/wave
}
```

### 미구현 필드 (v15에 있던 것)
- `expiresAt` — 자동 계산 가능하지만 저장 안 함
- `location` — 익명성 강화 + R2 없어서 보류
- `status` — emphasis/crowd는 *읽기 시 계산* (저장 X)
- `pulsePhase` — Output에서 hash로 생성

## 6. 시트 6 (Edge Cases) — v16 갱신

| Case | v15 | v16 상태 |
|---|---|---|
| A. 5분 이내 재태깅 | 임시 저장 복구 | localStorage draft에 자동 저장됨. 5분 제한은 없음. *지우기 버튼*으로 수동 |
| B. 5분 초과 재태깅 | 새 세션 | 5분 제한 없음. 마지막 draft 그대로 |
| C. 도킹 후 서버 응답 실패 | 3회 재시도 | Firestore 작성은 자체 재시도 + Long-polling. UI에 *재시도 알림*은 미구현 |
| D. 외벽 11개 초과 | 가장 오래된 페이드 | 24개 한도, 7일 자동 필터. 별도 페이드 알고리즘 없음 |
| E. 1초 미만 도킹 후 분리 | emphasis 직행 후 5초 페이드 | *해당 없음* (단일 송신) |
| F. 네트워크 끊김 | 로컬 캐시 | Firestore offline 캐시 (자동) |
| G. Server↔Output 통신 끊김 | 마지막 11개로 계속 | *해당 없음* (Server 없음). Firestore 끊기면 timeout 에러 |
| H. Input app 도킹 신호 미수신 | "다시 시도" 노출 | 시뮬 버튼이 대신함 |
| I. Output 부팅 시 Server 미작동 | 검정 화면 + 재시도 | `/wall`의 timeout 에러 화면 + "데모 모드로 보기" |
| J. 2단계 도킹 누락 | 30초 후 안내 | *해당 없음* (1단계 + 시뮬 버튼만) |
